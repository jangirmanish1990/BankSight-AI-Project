## Overview

**BankSight AI** is an Agentic Banking Analytics Assistant — a conversational AI system that takes
natural language queries, routes them through specialised subagents, queries structured banking data
(credit card spend, churn, customer scores, employee performance), generates insights, and delivers
automated Excel/PDF reports and Power BI-ready CSV exports.

The project demonstrates a hybrid of traditional data analytics (SAS/SQL domain expertise) with
modern agentic AI capabilities (Claude Code, subagents, custom slash commands, skills).

**Target users:** Banking analysts, data leads, and BI teams who want instant insights without
writing SQL manually.

## Architecture

BankSight AI uses a **multi-agent orchestration pattern**. The main Claude Code agent receives a
natural language query, determines intent, and delegates to the appropriate specialised subagent.
Each subagent runs in an isolated context window and returns structured output back to the
orchestrator, which then assembles the final response and triggers report generation.

```
User Query (natural language)
        │
        ▼
┌─────────────────────┐
│  Orchestrator Agent │  ← reads CLAUDE.md, routes intent
│  (Claude Code main) │
└──────────┬──────────┘
           │
    ┌──────┴───────┐
    │              │
    ▼              ▼
┌──────────┐  ┌──────────────┐
│ SQL      │  │ Analysis     │  ← runs in isolated context
│ Agent    │  │ Agent        │
└──────────┘  └──────────────┘
    │              │
    └──────┬───────┘
           ▼
    ┌──────────────┐   ┌──────────────┐
    │ Report Agent │   │ Anomaly      │
    │ (Excel/PDF)  │   │ Detector     │
    └──────────────┘   └──────────────┘
           │
           ▼
   dashboard/*.csv  →  Power BI refresh
   reports/*.pdf    →  delivered to user
```

### Key Components

**1. Orchestrator (Claude Code main session)**
The entry point for all user queries. Reads intent from natural language, selects the right
subagent(s) to invoke, passes structured context, and assembles the final response. Defined by
this `CLAUDE.md` file and the slash command definitions in `.claude/commands/`.

**2. SQL Agent (`agents/sql_agent.py`)**
Receives a natural language query and converts it to a valid SQLite SQL statement. Queries
`banking_mock.db` and returns raw results as a pandas DataFrame. Knows the full schema of all
4 tables (customers, transactions, loan_emi, employee_performance). Never modifies data — read
only.

**3. Analysis Agent (`agents/analysis_agent.py`)**
Receives the raw DataFrame from the SQL Agent and generates a human-readable insight summary.
Identifies key trends, top/bottom performers, and notable patterns. Returns structured text
suitable for displaying in the Streamlit UI or embedding in a report.

**4. Report Agent (`agents/report_agent.py`)**
Receives the analysis output and generates two deliverables automatically:
- An Excel file (`.xlsx`) saved to `reports/` using openpyxl
- A PDF summary saved to `reports/` using reportlab
- A CSV snapshot saved to `dashboard/` for Power BI refresh

**5. Anomaly Agent (`agents/anomaly_agent.py`)**
Scans transaction data for spend spikes (amount > 2× monthly average for that customer) and
flags unusual churn patterns (e.g. sudden drop in NPS + missed EMI in same cycle). Sets the
`is_anomaly` flag and returns a prioritised list of at-risk customers.

**6. Streamlit UI (`app.py`)**
The user-facing chat interface. Renders agent responses, metric cards (churn count, avg spend,
NPS), and download buttons for generated reports. Calls the Anthropic API and orchestrates
the agent pipeline on each user message.

### Development Pattern

BankSight AI follows **spec-driven development**. Before writing any code, read the relevant
spec file in `.claude/specs/`. Before running any agent, Claude Code reads this `CLAUDE.md`
to understand project context, schema, and conventions.

**Rules Claude Code must follow in this project:**

1. **Never modify `banking_mock.db` directly** — all data changes go through `setup_db.py`
2. **Never expose `customer_id` or `name` in reports** — anonymise before output
3. **All agents are read-only** — SQL agents use `SELECT` only, no `INSERT/UPDATE/DELETE`
4. **Always save outputs to the correct folder:**
   - Excel/PDF → `reports/`
   - CSV exports → `dashboard/`  (for Power BI refresh)
5. **Log every agent run** — print which agent ran, query received, rows returned, time taken
6. **If SQL query returns 0 rows** — Analysis Agent must return a friendly "no data found"
   message, not an error
7. **Anomaly threshold** — flag transactions where `amount > 2 × customer's monthly average`
8. **Churn rate benchmark** — dataset has 21% churn (681/3225). Flag if monthly rate exceeds 25%
9. **Currency** — all monetary values are in INR (₹). Never display as USD
10. **Date format** — always display as `DD-MMM-YYYY` (e.g. 07-May-2026) in all reports

## Important Notes for Claude Code

- The primary data store is `data/banking_mock.db` — always connect via SQLAlchemy or sqlite3
- The `customers` table is the central table — all other tables join on `customer_id`
- Churn analysis is the most important use case — prioritise accuracy on `churn = 1` queries
- When generating reports, always include: query date, total records analysed, and data range
- The `.claude/skills/churn-analysis/SKILL.md` skill should be loaded for any churn-related query
- Power BI connects to `dashboard/*.csv` files — never delete or rename these files mid-session

## Agent Thresholds & Benchmarks
(All agents must reference these values)

- Churn baseline: 21% (681/3225) — alert if exceeds 25%
- NPS range: 0–10 scale (not 0–100)
- Currency: INR (₹) only — never USD
- Date format: DD-MMM-YYYY (e.g. 07-May-2026)
- Anomaly threshold: amount > 2× customer monthly average
- Churn risk combo: NPS ≤ 3 + missed EMI in same month
- High risk segment: Basic + credit_score < 500
- Row limit per query: 100 (unless user specifies otherwise)
- Report save path: reports/ (Excel/PDF), dashboard/ (CSV)
- Google Drive folder: BankSight AI/Reports/[YYYY-MM]/
